Filename Translation


         The OPD and QL versions of Quill, Abacus, Archive and Easel
         have a different filename extension format.  OPDCOPY will
         automatically change the filename during the copying process
         to allow for this.  The extensions concerned are as follows:


                                                   QL      OPD

              Quill Save Files                   _doc      .doc

              Archive Database Files             _dbf      .dbf

              Archive Program Files              _prg      .prg

              Archive Screen Files               _scn      .scn

              Abacus Save Files                  _aba      .aba

              Easel Save Files                   _grf      .grf

              Export Files                       _exp      .exp

              Listing Files                      _lst      .lst

          


         Duplicate File Names


         A little care needs to be taken with filenames as OPD
         filenames are limited to 12 characters, and filenames longer
         than this are truncated by the OPD Driver software. 


         If OPDCOPY finds a file already exists on the output tape,
         then a message to this effect will be displayed, and you will
         be asked if you want to overwrite it.  If you reply NO, then
         the copying of the file named will be abandoned. An option of
         ALL is also available to allow any further duplicates to be

         overwritten without further checking.



         Duplicate Cartridge Names


         With OPD formatted cartridges, it is not possible to tell
         apart different cartridges with the same medium name. You
         should NOT therefore use the same Medium name for your live
         and backup copies of cartridges.


         The reason for this is that if you inadvertently take a
         cartridge out of the drive while a program still has a file
         open, and replace it with another cartridge with the SAME
         medium name, then the OPD Device Driver is unable to detect
         that the cartridge has been changed.  This means it will
         assume that the sector map held in memory is still valid.  As
         a result any attempt to write to that cartridge will almost
         certainly corrupt it beyond repair.


         This is in contrast to the QL microdrive software which
         (because of the difference in cartridge formats) is  
         normally able to tell apart two cartridges even if they DO
         have the same medium name.

         Warning messages regarding file compatability


         If you attempt to copy a file that the OPDCOPY program
         recognises as being of a type that is likely not to be
         compatible with the destination machine, then a warning
         message will be output of the form


              FILE: filename

              Warning Message

              COPY FILE:  YES  NO


         where the Warning message is one of those listed below. You
         must then decide if you still wish the file to be copied.

         MACHINE CODE FILE


              The are enough differences between the OPD and QL that

              machine code files are not normally portable between the

              two machines.


              WARNING: An attempt to run a converted machine code file

              can cause the system to crash, so do not copy such files

              unless you are certain you understand the way in which

              the Operating System interfaces used by machine code

              programs differ on the two machines.

                        Drive Level Compatability


         The OPD microdrive handling system appears to be more
         sensitive than the QL one, in that the failure rate in
         exchanging data on microdrive between machines is higher.


         If you find that you get READ FAILS when you attempt to read
         cartridges on the OPD, then this can often be avoided by
         using cartridges that have been formatted on the target OPD,
         instead of doing the formatting on the QL.

         4. USING THE OPD DEVICE DRIVER FROM SUPERBASIC


         The OPD Device Driver can be used directly without using the
         OPDCOPY program.  This allows Superbasic users to work
         directly with OPD formatted tapes.



         Loading the OPD Device Driver


         If you have booted up with the OPD/QL Interchange System 
         cartridge in Drive 1, then you will have a menu displayed
         giving the options available.  You merely need to select
         OPDLOAD to load the OPD Device Driver.


         Alternatively, you can load the OPD Device Driver by using
         the Superbasic command:


              EXEC_W xxx1_OPDLOAD


         where xxx is MDV for microdrive, or FLP for floppy disc.


         If the load is successful then the messages

              OPDLoad: Serial Number nnn

              OPDLoad: INITIALISED

         will be displayed.  


         If the load fails for any reason, then the message 

              'OPDLoad: ABANDONED'

         will be displayed, preceded by a message indicating the
         reason.



         Unloading the OPD Device Driver


         You can unload the OPD Device Driver at any time if you no
         longer require to use it. If you have the Menu supplied with
         the OPD/QL Interchange system displayed (or reload it by
         using the command LRUN xxx1_BOOT where xxx is the device
         containing the OPD/QL Interchange software), then you merely
         need to select the OPDEND option.


         Alternatively, You can unload the OPD Device Driver by using
         the SuperBasic command:


              EXEC_W xxxn_OPDEND


         where xxxn is the device and drive holding the OPD/QL
         Interchange software.


         The OPDEND program will wait for any outstanding operations
         on the microdrives to finish, detach the OPD Driver from
         QDOS, and release the memory that it had been using. It will
         then display the message

               'OPDEnd: COMPLETED'

         Using the OPD Device Driver


                            OPD/QL INTERCHANGE SYSTEM
                           USER GUIDE             Page 5 of 15
         Once the OPD Device Driver has been loaded, then, you will be
         able to open and close files in the normal way from 
         Superbasic, using OPD instead of MDV when you wish to refer
         to OPD formatted cartridges.


              e.g. FORMAT OPD2_opddata

                   OPEN #3,OPD1_testfile

                   INPUT #3,data$

                   PRINT #3,data$

                   CLOSE #3

                   DIR OPD2_


         If you have one of the QL toolkit packages, then you can also
         use the commands it provides with the OPD Device Driver.  


              e.g. WSTAT OPD1_

                   RENAME OPD1_opdfile1 TO OPD1_opdfile2

                   VIEW OPD1_opdfile

                   

         You can mix the use of QL and OPD formatted cartridges as 
         you still use MDV to refer to QL formatted cartridges.


         To handle the fact that you may by accident use MDV when you
         meant OPD, or OPD when you mean MDV, a new error message has
         been introduced.  It will occur when the cartridge in a drive
         has a format that is incompatible with the Device type you
         have specified.  This message will be:


                   Wrong format in drive n


         where n identifies the drive concerned. This message could
         also occur if you put an unformatted, or extremely badly
         corrupted, cartridge into the drive.

         Compatability with Other Standard Software


         Applications that use the QL Filing System interface will
         normally work unchanged with the OPD Device Driver.  Any

         various fast copy and cartridge recovery programs) will fail.

         5. USING THE OPD DEVICE DRIVER WITH THE PSION PACKAGES


         The recommended method of transferring QUILL, ABACUS, ARCHIVE
         and EASEL files between OPD and QL formats is as follows:


              a) If starting with OPD formatted cartridges, then use

                 the OPDCOPY program (covered in Section 3) to copy

                 the files onto a QL formatted cartridge.


              b) Run Quill, Abacus, Archive or Easel using QL

                 formatted cartridges.


              c) When you have finished, use the OPDCOPY program to 

                 copy your files from the QL formatted cartridge to

                 an OPD formatted cartridge.


         The remainder of this section covers the factors you have to
         consider if you are going to attempt to use the OPD Device
         Driver directly with Archive, Abacus or Easel instead of
         using the OPDCOPY program.


         NOTE. You cannot work directly with Quill files - see the

               section on Data File differences on the next page.




         Filename Extensions


         There is a difference in the format of filenames between the
         OPD and QL versions of the Psion packages. The extensions on
         the QL start with underscore (e.g. file_doc) while on the OPD
         they start with full stop (e.g.  file.doc). You will have to
         rename the OPD files to their QL equivalents before you can
         use the QL versions of the Psion packages. If you do not, you
         will get 'file not found' errors reported.




         Memory Usage


         The OPD Driver code requires approximately 5K of store, plus
         1K for each drive used in OPD mode. This can affect the
         capability of running the Psion packages. Without extra
         memory it is not practical to run version 1.0 of these
         packages.  With extra memory they can normally be used (but 
         you should upgrade to version 2 anyway because of the vast
         difference in performance). 


         Version 2.0 or later of the packages can be run on a basic
         128Kb QL. The programs will have their free store reduced as
         mentioned above.  However, as the QL normally has more free
         store than the OPD you rarely find that this matters.

         Data File Differences


         The OPD Device Driver handles the differences in the
         Microdrive formats of the OPD and QL, but it cannot handle
         the situation in which the actual data within files is
         different.  As a result it is not possible to use all file
         types directly.


         The File Types that CAN be handled directly are:


              Abacus Save Files                  (_aba)

              Archive Database Files             (_dbf)

              Archive Program Files              (_prg)

              Archive Screen Files (Version 2)   (_scn)

              Easel Save Files                   (_grf)

              Export Files                       (_exp)

              Listing Files                      (_lis)


         The File Types than CANNOT be handled directly are:


              Archive Screen files (Version 1)   (_scn)

              Quill Save Files                   (_doc)

                   (Quill will appear to work as it will give a

                    Loading.... message, but will then fail with

                    the message 'Not a valid Quill File')


         There are additional file types that are available on the

         QL implementations that have no equivalent in the OPD

         implementation of the same package:


              Archive Object Program             (_pro)

              Easel Screen Dumps                 (_pic) 



         Character Set Differences


         This is covered in detail in the OPD/QL Basic Conversion
         Guide.



         Drive Usage


         The Psion packages are normally configured to expect device
         type MDV. Unless you reconfigure them to expect an OPD device
         type, then you must take care not have a microdrive cartridge
         of the wrong format loaded at any time.

         APPENDIX  QDOS FILING SYSTEM INTERFACE EXTENSIONS


              ******************************************************

              **   This Appendix is only relevant if you intend   **

              **   to access the facilties of the OPD Device      **

              **   Driver from your own Assembler programs.       **

              ****************************************************** 


         The OPD Device Driver supports all the standard QDOS Filing
         Systems calls as documeneted in the Sinclair QL Technical
         Guide.


         In addition, the additional calls supported by the Tony Tebby
         Toolkit (available from QJUMP) are also supported.  These
         calls are also built into the majority of Disk interfaces
         available on the QL.  The detail of these extensions is as
         follows:-



         IO.OPEN   Trap #2   D0=$01


         Open in overwrite mode (D3.L=3) is supported.


         The file creation date is set for OPD NEW and OPEN OVERWRITE
         modes.



         IO.CLOSE  Trap #2   D0=$02


         The file header will have the update date field set if the
         file was changed in any way.



         FS.RENAM  Trap #3   D0=$4A


         This call allows a file to be renamed.


              Call Parameters:         A0  Channel Id

                                       A1  Address of a string which

                                           is the new name.



         FS.TRUNC  Trap #3   D0=$4B


         This call truncates the file to the length at which the file
         pointer is currently positioned.



              Call Parameters:         A0  Channel Id
