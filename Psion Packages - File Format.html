PSION PACKAGES - FILE FORMATS


         Introduction


         The following document gives information on the file formats
         used within the Psion packages.  This information is not
         supplied by Psion, but instead is empirical, gleaned from
         examining the files and various snippets from magazines.


         The files are listed in the alphabetical order of the
         extensions used to identify the file type.

                   FILE DATA


                   A single record is always held as contigous data,

                   but the end of one record does not necessarily mean

                   the start of the next.  Instead records are located

                   by means of the File Index table.


                   Fields are not in strict create order.  Instead all

                   numeric fields are grouped at the start of a

                   record.


                   Numeric fields are held as 6 byte floating point

                   numbers.


                   Strings consist of 2 byte length field followed by

                   text.

         Export Files (_EXP)


                   ASCII characters.  Each line is terminated by CR LF

                   and End of file indicated by CTRL/Z ($1A).



         Easel Save files (_GRF)


                   DATA AREA


                   Initial data consists of ASCII character strings.

                   The data is held in standard DIF (comma separated

                   values with text in quotes) format. Each line is 

                   terminated with CR,LF.


                   End of data area appears to be ESC ($1B).


                   The first line contains "label$", and then the

                   names of each set of figures.  Subsequent lines

                   apply to each period, and consist of the period

                   name followed by the corresponding values from each

                   set of figures.


                   CONTROL AREA


                   It is not clear how the remainder of the file

                   (which contains information such as Names, labels, 

                   format etc is constructed.


                   Text entries ??

                             1 byte    length

                             1 byte    direction (0=vertical

                                                  1=horizontal?)

                             n bytes   text data



         Listing Files (_LST)


                   Simple ASCII character data.  Printer control

                   characters will be interspersed with the data.



         Archive Program Files (_PRG)


                   Simple text file.  Logical length given in the

                   file header.  Lines finish with CR LF. File 

                   finishes with NAK ($1A).
