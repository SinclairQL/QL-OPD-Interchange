INTRODUCTION


         This document summarises the stages that have occurred in the
         development of the OPD/QL Interchange System. 


         It is intended to serve a dual purpose. The first is to act
         as documentation of when facilities were introduced and at
         what stage bugs were discovered and (hopefully) cured. The
         second is to provide a document from which the relevant pages
         can be issued to let existing users what they will gain if
         they apply for a new release.



         RELEASE 1           Never Issued


         This was a test only version that consisted mainly of
         SuperBasic with a few machine code subroutines.


         Its purpose was to prove that it was possible to write
         software that would allow the QL to read and write OPD
         formatted Microdrive cartridges.




         RELEASE 2.0         Issued on 20th October, 1985


         This was the first genuine version of the software.  It was
         only released on a test basis.


         New Facilities/Changes


         1. The original trial system re-written completely in machine

            code, and redesigned to be used by users without detailed

            knowledge of QL Device Drivers.


         2. The documentation re-written from scratch to reflect the

            new design and program facilities to be provided.


         3. The load process changed so that you can load and unload

            the OPD Device Driver at any time.


         4. The Backup process changed to let you make any number of

            backup copies.


         5. The OPDTAPE diagnostic program added to the software.

         RELEASE 2.1         Issued on 1st November, 1985


         This version was only released on a test basis


         New Facilities/Changes


         1. Support for the QL Toolkit RENAME and TRUNC commands

            added.


         2. The restriction that meant that MDV5 to MDV8 could not be

            used with the OPD Device Driver removed.


         3. The OPDCOPY program allows backup copies of the issued

            software to be made to Floppy Disc as well as Microdrive.


         4. The OPDCOPY program provides an option for the automatic

            conversion of filename extensions used by Psion programs.


         5. The OPDCOPY program provides an option for the translation

            of some of the character set differences.


         6. The OPDCOPY program no longer automatically overwrites a

            file if it already exists.  The user is instead prompted

            for the action to be taken.


         7. The OPDTAPE program now has an option to allow you to list

            the current block displayed to a printer or file. 


         8. The OPDTAPE program now allows you to ask for blocks

            by file number as well as absolute block number.

         VERSION 2.2         Issued on 1st December, 1985


         New Facilities


         1. The OPD Device Driver read/write process speeded up.


         2. OPDCOPY enhanced to make data and file name translations

            automatic when the filename allows the file to be

            identified  as being a Quill, Archive, Abacus, Easel or

            Basic file.   


            The prompt for name translation has therefore been

            removed, and that for Data Translation now only applies

            to files that cannot be identified by the file name.

            

         3. OPDCOPY enhanced to make an automatic decision on the Data

            Translate requirement when the file type can be determined

            from the file name.


         4. OPDCOPY given an option to allow the User to select

            whether any particular file should be copied.  


         5. The OPDTAPE program renamed to OPDDIAG, and the format of

            blocks printed/dumped improved.


         6. Extensive changes made to the documentation to improve

            its comprehensiblity, and to include the facility changes

            mentioned above.


         7. The OPD/QL Basic Keyword checklist added to the issued

            documentation.



         Bugs Cleared


         1. Use of the Data Translate option would cause the OPDCOPY

            program to fail.


         2. The Filename translate option in OPDCOPY would not work

            correctly for all file types.

         RELEASE 2.3         Issued on 10th January, 1986


         This was the first version of the softwae that appeared to
         encounter no difficulties in normal use.


         New facilities/Changes


         1. The OPDCOPY program will now output an warning message if 

            you try to convert QL files that are incompatible with

            the OPD XCHANGE implementation.


         2. A number of Error conditions were not being handled 

            correctly by the OPD Device Driver. This has been

            rectified.


         3. Changes made to OPDCOPY, OPDDIAG and BOOT to improve the

            presentation.   


         4. The handling of errors within the OPDCOPY program has

            been substantially improved.


         5. Documentation updated to reflect new facilities, and also

            the User Guide partially re-written in the light of

            comments from existing users.



         Bugs cleared


         1. The OPDCOPY program would fail when trying to copy a

            second cartridge in the same run.


         2. The OPDCOPY program would fail when copying files large

            enough to make it display the 'COPYING' message.

         RELEASE 2.4         Issued on 20th February, 1986


         New facilities/Changes


         1. A complete re-write of the code used to check for the

            wrongly formatted cartridges has been done.

            

         2. The OPDCOPY program will now output a warning message if

            you try to copy machine code files in either direction, or

            if you try to copy an OPD file type that has no equivalent

            on the QL (e.g. a Store Save file).


         3. The User Guide restructured to make it clearer that the

            OPDCOPY program can be used without understanding how to

            use the OPD Device Driver from Superbasic.



         RELEASE 2.5         Issued on 10th March, 1986


         New Facilities/Changes


         1. The OPDCOPY program will also copy the Medium Id for

            QL to QL copies if copyright symbol used as file name.

            This option is NOT documented.


         Bugs Cleared


         1. The OPDCOPY program was failing to produce a warning

            message when copying non-data file types (machine code,

            OPD save files, OPD catalogue files).


         2. The OPD Device Driver was failing to produce a 'Wrong

            format in Drive n' message when MDV was used for tapes

            in OPD format.  The same fault could also occasionally

            cause the system to crash, leaving the drives spinning.


         3. The OPDCOPY program was producing erroneous warning

            messages when in the 'Copying' mode of operation.


         4. The OPDCOPY program was not copying all File Header

            information correctly.



         RELEASE 2.51        Issued on 26th March 1986


         New Facilities/Changes


         1. The load process changed slightly (OPDLOAD run as a slave

            job by OPDCOPY and OPDDIAG rather than being included as

            an inbuilt subroutine).


         2. The OPDCHANGE program added.

         RELEASE 2.6         Issued on 21st July, 1986

         

         New Facilities/Changes


         *  The OPD DRIVER has had the routines for reading and

            writing improved.  There were slight differences in the

            timings used by the OPD Driver software on the QL compared

            to a real OPD.  This change should reduce the chance of

            cartridges written on the QL later getting an I/O Error on

            the OPD.


         *  OPDCOPY has had a number of cosmetic improvements made to

            the presentation of data and handling of errors has been

            improved to give more meaningful message.


         *  OPDCOPY now does Data Translation on the Currency symbol

            for Abacus Save files.


         *  OPDCOPY now does Data Translation on Archive Screen files.


         *  OPDCOPY has had the mechanism used for Data Translation

            improved. This means that it is faster in normal use, and

            also that Foreign Language conversions can be allowed for.


         *  OPDCOPY has improved the way it Loads files into memory.

            The result is that the load is less likely to fail, but 

            can take longer if a significant number of re-tries is

            needed to access the data. A warning message will be

            output if recovery takes place as this may be due to the

            use of badly worn cartridges.


         *  OPDCOPY now allows you to answer prompts for the drive

            number by pressing the corresponding numeric key as well

            as by the previous multiple selection method.


         *  OPDCHANGE program re-named OPDCONFIG.  Option to set

            highest Drive Number added.


         *  OPDBASIC program added to the software.


         *  A Major re-write of the OPD/QL BASIC KEYWORD SUMMARY

            document done, and the result re-titled as the OPD/QL

            BASIC CONVERSION GUIDE.


         *  OPDDIAG and the OPDREF document no longer issued as

            standard.  Available upon request to existing customers

            taking an update.  This step was taken because the

            software no longer fitted onto a single cartridge, and the

            vast majority of users did not use these options.

         Bugs Cleared


         *  The OPD DRIVER could erroneously output the "Wrong Format

            in Drive n" message.


         * The OPD DRIVER would not clear the copy of a Catalogue held

           in memory when a cartridge was reformatted.  If the

           cartridge was reformatted with the same Medium name, then

           this could result in files still being shown as present on

           the cartridge even though they were not really there.


         * The OPD DRIVER was not supporting the QDOS traps for

           setting the file pointer (FS_POSRE and  FS_POSAB)

           correctly.


         * The OPD DRIVER was not resetting the Sector Map in the

           catalogue correctly when a file was deleted.  This resulted

           in some of the space released not being made available.

         RELEASE 2.7         Under Development - Not Yet Released


         New Facilities/Changes


         * OPDBASIC program altered so that those with Supercharge

           could use it on the program with no alterations required.

          

         * OPDCONFIG has had an option to change the Data Translation

           (default) English one to be converted correctly.



         Bugs Cleared


         * OPDCOPY would ignore files loaded after a 'Load Fail'

           has occurred.  Subsequent files would be loaded into

           memory, but not written to the output device.


         * OPDCOPY would not output an error message if a copy was

           abandoned due to 'drive full'.


         * OPDCOPY would not output an error message if a format

           failed, but would merely offer a chance to try again.

         ENHANCEMENT IDEAS


         This section is a list of ideas for incorporation in future
         versions of the software.  They have mainly arisen as the
         result of feedback from existing users.  There is no
         guarantee that they will ever be incorporated.


         OPD/QL Toolkit


         The current OPD/QL Interchange software is aimed at general
         use.  There could be a use for a lower level package (to be
         known as the OPD/QL Toolkit) to complement the OPD/QL
         Interchange software. The toolkit is envisaged as initially
         consisting of the following major components.


                   - The OPDDIAG program (which would be removed

                     from the OPD/QL Interchange package).


                   - Extensions to SuperBasic to allow sectors on

                     OPD format cartridges (and QL ones) to be

                     accessed directly from within Basic programs.


                   - A program that would provide facilities for

                     recovering files from corrupt OPD cartridges.

                     It would be able to recover files even if the

                     volume map or catalogues were corrupt. In the

                     case of individual sectors within a file being

                     corrupt, the remainder would still be recovered.


                     

         OPD DRIVER


         *  Get the OPD Driver Physical layer to run independently of

            the Access Layer. This would give a further performance

            boost.

            NOTE.  The software is designed to run like this, but

                   problems have been encountered in getting it to

                   work in this mode.


         OPDCOPY


         *  Check the amount of free space when copying in unselective

            mode and give warning message if insufficient.


         *  Check before copying each file if it will fit (NOTE must 

            also allow for an existing copy).


         *  Wild Card copy option to select files copied.


         *  Allow for the addition/removal of file-name prefixes

            (directory information?). This will help with disc based 

            systems.

         


                            OPD/QL INTERCHANGE SYSTEM

         *  When doing MDV to MDV or OPD to OPD, and all files are to

            be copied, then do a physical level 'express' copy.



         OPDCONFIG


         *  Allow the timings of the FORMAT routine within the OPD

            Device Driver to be tailored to a specific QL/OPD pair.



         Known Bugs (not yet cleared)


         * The OPD Device Driver will not work in conjunction with

           Eidersoft's ICE system.  Any attempt to use ICE to get a 

           directory of an OPD cartridge will 'lock' the QL and mean

           that the system has to be RESET.
