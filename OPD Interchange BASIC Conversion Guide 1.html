         1. INTRODUCTION


         There  are  great  similarities  between  the  Sinclair     QL
         SuperBasic  and  ICL  OPD Basic.  You can therefore relatively
         easily  transfer many programs between the two machines. There
         are  also  differences, so although programs do not need to be
         completely  rewritten  during  the  conversion,  they may well
         need a number of minor changes.


         This  Conversion  Guide is structured in four main sections as
         follows:



           -  A  Discussion  of the generic differences between the OPD
              and QL that affect Basic Programs.



           -  Instructions  on using the program OPDBASIC. This program
              is  used to help identify areas of programs that may need
              changing during the conversion.



            - A consolidated alphabetic list of the Keywords used by:

                        OPD Basic

                        QL SuperBasic

                        QL Toolkit 

                        QL Super Toolkit 2


              The  last  two  have  been included as many QL users have
              one  or  the  other. When converting from the QL, you may
              have  used  the  features they offer, and when converting
              the  other  way  you need to know if there is a potential
              conflict  between  the  names  used  in your program, and
              those used by either of the Toolkits.



            - Miscellaneous  notes on items affecting the conversion of
              programs.

         2. GENERAL CONVERSION GUIDLINES


         The  OPD  version  of Basic was based on that used for the QL.
         The  commands  that  are  not  related  to  specific  hardware
         features  are  therefore  identical  on  the  two    machines.
         Differences  were  introduced to take account of the fact that
         the  two  machines have different hardware, operating systems,
         and User Interfaces.


         This  section  therefore gives you some idea of the areas that
         can  cause  programs  to  need  alteration to run on the other
         machine.



         Program Size


         The  QL  has  Basic  permanently  resident in ROM.  On a basic
         128Kb  QL  there  is  typically  85-90Kb free after the memory
         required by the operating system has been deducted.


         On  the OPD, Basic has to be loaded off Microdrive. Basic also
         allocates  memory  to  save  the  screen  contents in case the
         START,  RESUME  or  REVIEW  keys are used.  The result is that
         there  is  only  about 30-35Kb free for the program.  This can
         be  even  further agravated by the fact that one tends to keep
         items such as a Telephone Directory in the OPD's memory.


         Memory Allocation


         The  OPD  and  QL  have  different methods of memory handling.
         Unlike  the  QL method, the OPD system allows for machine code
         programs to be moved in memory even after they are loaded.


         This  is  visible when one uses the OPD housekeeping option to
         look  at  memory  usage.    The  items in memory are not given
         addresses  (as  these  might  change) and the size is given in
         blocks  (which  are the OPD unit of memory allocation) each of
         512 bytes in size.


         Screen


         The  OPD  has a maximum of 480 x 240 pixels available to Basic
         because  some  space  at  the bottom of the screen is reserved
         for  use  by  the  'Noticeboard'.    In  addition  only    the
         equivalent to the QL high resolution mode is available.


         Graphics


         The  OPD  does  not  support  most  of  the  graphics  related
         commands  of  the  QL.  The reason for this is the memory that
         would  be  needed  to support the saving of screens containing
         graphic  data  when  the START, RESUME or REVIEW keys are used
         on the OPD.



         Jobs

         The  OPD  does  not  allow  machine  code  jobs to be created,
         started  or  controlled  from  Basic. This is instead done via
         the Menu system.

          


         Character Set Differences


         There  is  a  difference in the character sets used by the two
         machines.    For  many  purposes  these  differences  can   be
         ignored, but this is not always the case.


         The  character  set can be considered as divided into two main
         sections:-


           a) The  characters  which  are  part  of  the  normal  ASCII
              graphics  set (those having internal values up to decimal
              127,  hex  7F). For this range of values the two machines
              largely correspond.


              There  are  a few characters which are part of this range
              on  of  the  machines which do not have the same internal
              value on the other machine.  They are the characters:

                             Pound

                             Hash

                             Copyright

                             Tilde (backward quote)


              The  OPDCOPY  program  contains  special Data Translation
              facilities  to  handle the conversion of these characters
              as  part  of  the  copy process (as long as a file can be
              identified as containing printable data).


           b) Characters  having  values  above  the  normal  ASCII set
              (internal  values of decimal 128, hex 80, upwards). These
              are completely different on the two machines.


              The  complete  character  sets for each machine are given
              in  the  Concepts  section  of  the Basic manuals for the
              respective machines.

         3. THE OPDBASIC PROGAM


         Overview


         This  program  is  intended  to  act  as  an aid to converting
         programs  between  the  OPD  and  QL.  It will scan through an
         existing  Basic  program  identifying  statements  that  might
         cause problems.


         If  the  file is on an OPD format cartridge, then the OPDBASIC
         program  can  still  handle  it  correctly  as long as the OPD
         Device  Driver  is  already  loaded.  However, before you edit
         the  program  on the QL you should first convert the file to a
         QL  format  file  using  the OPDCOPY program.  If the filename
         does  not  end with _bas then the Data Translate prompt should
         be set to YES.



         Running the Program


         The program is run by using the Superbasic command:


                   LRUN xxxn_OPDBASIC


         where  xxxn  identifies  the  device  and drive from which the
         program is to be loaded (e.g. MDV1).


         The  program  will  then  provide  on  screen  prompts for the
         information it requires from the user.



         Listing Messages


         The  program  will identify any statement that may not convert
         correctly with a message of the form


         Line nnn:  xxxx:  Verb:  Text           where


         nnn  identifies the Basic Line Number


         xxxx is the seriousness categorised as follows:


              ERRORS    Commands  that  are  not  supported  by    both
                        machines.

              WARNINGS  Keywords   that   are   implemented    slightly
                        differently on the two machines.

              CAUTIONS  Keywords  that  are  used  by  one  of  the  QL
                        Toolkits.  They will only cause problems if the
                        QL DOES use one of the Toolkits. 


         verb  is the Keyword concerned


         text  gives more information concerning the problem area.

         4.  KEYWORD SUMMARY


         This  section  is a complete list of the Keywords used in both
         versions  of  Basic.  The  versions of Basic used as reference
         points are:


              OPD BASIC                Version 7.00

              QL  SUPERBASIC           Version JS


         The following conventions are used:


            - Where  an keyword is not available on both machines, then
              its origin is indicated by using:


                   OPD       OPD standard keyword

                   QL        QL Standard keyword

                   TK        QL Toolkit keyword

                   TK2       QL Super Toolkit keyword


           -  If  the  same  command  exists on both machines, but with
              difference  syntax,  then  this  is  indicated  using the
              following method to indicate syntax.


              UPPER case indicates fixed part of the command

              LOWER case indicates variable part of command


              [ ]  indicate the contents are optional

              { }  indicates the contents can be repeated


               For more information on the syntax of any command, refer
              to the Basic manual for the relevant machine.
