         1. INTRODUCTION


         In early 1984 Sinclair introduced the QL computer aimed at
         providing to the home user many of the facilities that at
         that time were considered to be only available on much more
         expensive business computers. At about the same time ICL
         entered a colaboration agreement with Sinclair Research to
         allow ICL to use the QL technology in a 'computer-phone'
         aimed at the Business marketplace.


         At the end of 1984, ICL launched its One Per Desk (or OPD)
         workstation into the Business marketplace. Since the OPD owes
         many of its personal computing aspects to the results of the
         colaboration agreement between ICL and Sinclair Research, it
         is not surprising they both use Microdrives to hold data
         files, and they both support the Psion software packages
         Quill, Abacus, Archive and Easel.  In addition, OPD Basic is
         a subset of the SuperBasic supported by the Sinclair QL.


         The OPD is intended for a different market to the QL, so
         there are also a significant number of differences between
         the facilities offered on the two machines. Those who have
         access to both machines will have a great interest in being
         able to easily exchange files between the two environments.

         The obvious medium to use is microdrive cartridges, but the
         differences between the two machines are significant enough
         to mean that the standard software provided will not let the
         QL read or write OPD formatted cartridges, and vice versa.


                   The OPD/QL Interchange System is specifically
         designed to support such interchange.  When the OPD/QL
         Interchange software is loaded into QL it gives the QL the
         ability to read and write files on OPD formatted cartridges
         as easeily as on QL formatted ones.  This means the QL is
         then able to work with cartridges that were either created on
         an OPD, or are later to be used on an OPD.  The commonest use
         of this facility is expected to be the transfer back and
         forth of Quill, Abacus, Archive and Easel files, and
         occasionally Superbasic programs, so the facilities provided
         make the transfer of such files as easy and foolproof as
         possible.


         The OPD/QL Interchange System consists of two major
         components:-


              a) The OPD Device Driver that is responsible for the 

                 actual reading and writing of OPD format cartridges.

                 It allows you to:


                   -  Read on the Sinclair QL, microdrive cartridges

                      that have been created on an ICL OPD.


                   -  Write on the Sinclair QL, microdrive cartridges

                      that can be used on an ICL OPD.


              b) The OPDCOPY program that provides:


                            OPD/QL INTERCHANGE SYSTEM
                           USER GUIDE             Page 3 of 15
                   -  A Menu driven interface for the fast copying of

                      complete microdrive cartridges. Options are

                      given to allow only selected files to be copied.


                   -  Automatic conversion of differences in the file

                      formats used by the OPD and QL versions of

                      Quill, Abacus, Archive and Easel.


                   -  An Option for applying data translation on text

                      files to handle the slight differences in the 

                      in the OPD and QL character sets.


         A number of other useful programs are also included.


              a) The OPDLOAD and OPDEND programs that allow the OPD

                 Device Driver to be loaded and unloaded without you

                 having to reset your QL.


              b) The OPDBASIC program will scan a Basic program

                 highlighting keywords for which the OPD and QL

                 implementations are not the same.


              c) The OPDCONFIG program allows the OPD/QL Interchange

                 software to be tailored to your particular QL

                 hardware configuration.


         Extensive documentation is supplied in Quill files.  It is
         broken into the following parts:-


              a) The User Guide (this document) which contains all you

                 need to know for standard use of the OPD/QL

                 Interchange System software. 


              b) The Basic Conversion Guide for those intending to

                 convert Basic programs between the QL and OPD.


              c) The Configuring Guide for those intending to change

                 some of the default options used by the software.

         2. MAKING BACKUP COPIES OF THE OPD/QL INTERCHANGE SOFTWARE


         Microdrive cartridges can become faulty with prolonged use,
         so it is advisable to make a backup copy of the original
         program cartridge containing the OPD/QL Interchange System
         software. The original should then be put in a safe place,
         and the backup copy used for loading the OPD/QL Interchange
         System in normal use. 


         The OPD/QL Interchange System program cartridge has a
         security mechanism to protect against unauthorised
         duplication.  If you attempt to use the normal Superbasic
         COPY command to backup the software, then you will find that
         any attempt to use such a copy with an 'UNAUTHORISED COPY'
         message.


         The mechanism for making valid copies that will run is built
         into the OPDCOPY program (covered in more detail in section
         3).  The option to make a backup will be offered whenever you
         have loaded the OPDCOPY program from the original issue
         cartridge.  You will NOT be given this option if you have
         loaded from a backup copy of the software. OPDCOPY will allow
         you to make as many backup copies as you like of the original
         cartridge, but the copies cannot be copied again.  All the
         copies will be identified when they load by the same Serial
         Number as the original master copy. 


         The backup process allows you to specify the following
         information:-


              a) The device type to which you are copying.


                 During the backup process the programs will be

                 re-configured to run off XXX1 where XXX is the device

                 name you selected.


              b) The drive number to be used


              c) Whether the target medium needs to be formatted

                 before use, and if it does the name you wish to give

                 it. If you merely press enter in reply to the medium 

                 name prompt, then the name will be copied from the

                 Source medium.


              d) Whether you wish to invoke File Selection during

                 the copy process to select which files are to be

                 copied. The files that must be copied for the

                 software to function correctly are:

                             OPDLOAD

                             OPDEND

                             OPDCOPY

                             OPDDRIVER

         3. THE OPDCOPY PROGRAM


         Overview


         The easiest way to use the  OPD/QL Interchange System is to
         use the OPDCOPY program to copy the OPD formatted cartridges
         to/from QL formatted ones at the beginning/end of sessions.
         The OPDCOPY program provides a simple-to-use interface for
         copying all or part of a cartridge.  You then work during a
         session on the QL formatted files in your normal manner.


         Both the input and output cartridges can be in either QL or
         OPD formats. When copying between QL and OPD formats special
         options are available to help in the handling of differences
         between the data contents of files in the two formats.


         This program also provides the ONLY method of making backup
         copies of the cartridge containing the issued OPD/QL
         Interchange System software (as covered in section 2).


         Loading


         If you have booted up with the OPD/QL Interchange System
         cartridge in Drive 1, then you will have a menu displayed,
         and you merely need to select the appropriate option.

         Alternatively, the program can be invoked directly with the 
         Superbasic command:


                             EXEC_W xxx1_OPDCOPY


         where xxx is the device from which you are loading (e.g.MDV).


         Parameter Selection


         OPDCOPY works by giving you multiple choices whenever
         possible, and allowing you to toggle between them, with the
         current choice being  highlighted. The program will initially
         highlight the most commonly used choice, but you can select
         one of the other options if you wish. The keys used to select
         from the options displayed are as follows:


              Right Arrow / SPACE      Toggle to the next choice


              Left Arrow               Toggle to the previous choice


              Down Arrow / ENTER       Accept the current choice


              Up Arrow / ESC           Return to the previous        
                                       question.


              Numeric Keys             Can be used as an alternative

                                       to the cursor keys when

                                       answering Drive Number prompts.

                                       

         You will be prompted for:


                            OPD/QL INTERCHANGE SYSTEM

         SOURCE:

              You must give the device type and drive number. For 

              device types of MDV and OPD the program will check that

              the microdrive cartridges in the drive is in a valid

              format for the choice you made.


         DESTINATION:

              You must give the device type and drive number, and

              whether it is to be re-formatted.   If it is to be re-

              formatted then the program will ask for the  name you

              wish to give it. Pressing only ENTER in response to the

              Medium name prompt cause the Medium name to be copied

              from the Source medium. If you specify no re-formatting,

              then for device types of OPD and MDV the format of the

              microdrive cartridge will be checked as being correct.


         CHANGE OPTIONS:

              This is used to select a second screen giving a number

              of options that can normally be defaulted.  If you reply

              NO, then the copy will start.


         If you change the CHANGE OPTIONS prompt to YES and press
         ENTER, then the following additional choices will be given.


         DATA TRANSLATION:

              This option can normally be defaulted (i.e. set to NO).


              It allows for the conversion of those characters that
              form part of the printable character set, but that have
              different internal values on the OPD and QL.  Using the
              standard translation tables the characters concerned are
              hash, pound, copyright and tilde. 


              As this conversion can only be done on files that
              contain printable characters, it will never be done on
              files that are identified by the file type as containing
              binary data.


              This option will be invoked automatically for file types
              that can be identified by the last part of the filename
              (regardless of the setting of the Data translate
              prompt).  These file types are:

                                                   QL       OPD

                   Basic Program Files           _bas      _bas

                   Quill Document Files          _doc      .doc

                   Archive Database Files        _dbf      .dbf

                   Archive Program Files         _prg      .prg

                   Archive Screen Files          _scn      .scn

                   Abacus Save Files             _aba      .aba

                   Easel Save Files              _grf      .grf

                   Export Files                  _exp      .exp

                   Listing Files                 _lis      .lis

              If you have files that will need this conversion option,
              but  whose filename does not finish with one of these
              endings, then you should change the Data Translate
              prompt to YES.

         FILE SELECTION:

              If this option is set to NO, then all files are copied.

              If it is set to YES, then before a file is copied, the

              name will be displayed and the User asked to reply YES

              or NO to the copying of that particular file. A reply of

              ALL is also avaialable to indicate that all the

              remaining files are to be copied.

              


         Formatting the Target Medium

         

         If you have asked for the target medium to be formatted then,
         when the formatting process starts, the current format and
         medium name (if recognised) will be displayed and you will be
         asked to confirm the format.


         If you reply YES, then the format will proceed.  If you
         respond NO, then you will be asked whether the copy should be
         retried with another target cartridge  (YES), abandoned (NO),
         or (if it is already in the correct format) the target medium
         used without formatting (USE).  



         The Copying Process


         While the program is running, the bottom half of the screen
         will be used to give messages concerning the progress of the
         copying process.

                                                                      
         The OPDCOPY program will take advantage of extra memory to
         speed up the copying process. The program supports two
         copying methods depending on the free memory. It may use both
         methods at different times during the copying of a cartridge
         as is appropriate.


              - The first and more efficient method loads complete

                files into memory (as many as possible), and then 

                saves complete files to the destination drive. When

                this method is being used, the program will display

                messages of the form

                   LOADING..Mdv1_filename

                   SAVING...Mdv2_filename


              - The second method is used to copy files that are too

                large to be loaded into memory.  The program will then

                copy them in sections. When this method is being used,

                the program will display messages of the form

                   COPYING...filename
